================================================================================
CORE CONCEPT
================================================================================

Whisper is a multi-layer voice-to-text system that can:
  1. IDENTIFY running CLI processes (Claude, Codex, etc.) using process scanning
  2. COMMUNICATE with them via direct stdin injection, clipboard, or named pipes
  3. OPERATE in multiple modes: interactive CLI, headless scripting, GUI, daemon

================================================================================
KEY IDENTIFICATION MECHANISM
================================================================================

HOW IT FINDS CLAUDE/CODEX:
  - Scans all processes: psutil.process_iter(['pid', 'name', 'cmdline'])
  - Checks if 'claude' or 'codex' appears in process command line (case-insensitive)
  - Stores as: "claude (PID: 9012)" in GUI dropdown
  - Each process has unique /proc/[pid]/fd/0 file descriptor for stdin

WHAT MAKES EACH INSTANCE UNIQUE:
  - Process ID (PID) - system-assigned identifier
  - Command-line arguments - full command string
  - stdin file descriptor - /proc/[pid]/fd/0 (each process has its own)
  - Environment variables - DISPLAY, WAYLAND_DISPLAY, etc.

================================================================================
THREE COMMUNICATION CHANNELS
================================================================================

1. DIRECT STDIN INJECTION (Most Direct)
   - Open: /proc/[pid]/fd/0
   - Write: transcribed text + newline
   - Effect: Text appears in target process as if user typed it
   - Used by: GUI application

2. CLIPBOARD (Fallback, Universal)
   - Wayland: subprocess.run(['wl-copy'], input=text)
   - X11: subprocess.run(['xclip', '-selection', 'clipboard'], input=text)
   - Effect: User CTRL+V to paste in any application
   - Used by: GUI application (always), Hotkey daemon

3. NAMED PIPE/FIFO (For Scripting)
   - Environment: WHISPER_TRANSCRIPT_FIFO=/tmp/whisper.fifo
   - Command: python whisper_cli.py --headless
   - Effect: Transcription written to named pipe
   - Used by: Headless mode, external scripts

================================================================================
EXECUTION MODES & RECOGNITION
================================================================================

HEADLESS (Scripting/Automation):
  Command: python whisper_cli.py --headless [--debug]
  Recognition: --headless flag, self.headless = True
  Behavior: Single cycle (record → transcribe → output), no prompts
  Output: stdout or FIFO only

INTERACTIVE (User CLI):
  Command: ./whisper OR python whisper_cli.py [--debug]
  Recognition: No headless flag, full REPL loop
  Behavior: Prompts user, ENTER to record/stop, emoji output
  Output: Console display

GUI APPLICATION (Advanced):
  Command: python whisper_gui.py OR ./launch_gui.sh
  Recognition: PyQt6 window with dropdown process selector
  Features: Process discovery, history, multi-threaded, clipboard mgmt
  Output: stdin injection + clipboard

HOTKEY DAEMON (System-wide):
  Command: sudo ./run_daemon.sh --debug [or systemd service]
  Recognition: Monitors /dev/input/event* for CTRL+ALT+R
  Behavior: Works in any application, copies to clipboard
  Output: Clipboard only

PROCESS DISCOVERY (from whisper_gui.py):
  for proc in psutil.process_iter(['pid', 'name', 'cmdline']):
      cmdline_str = ' '.join(proc.info['cmdline'])
      if 'claude' in cmdline_str.lower() or 'codex' in cmdline_str.lower():
          display_name = f"{cmd_name} (PID: {proc.pid})"
          self.available_processes[display_name] = (proc.pid, proc)

STDIN INJECTION (from whisper_gui.py):
  stdin_path = f"/proc/{self.selected_process_pid}/fd/0"
  with open(stdin_path, 'w') as stdin_fd:
      stdin_fd.write(text + '\n')
      stdin_fd.flush()

================================================================================
TYPICAL WORKFLOW - GUI WITH CLAUDE
================================================================================

1. Terminal 1: Start Claude
   $ claude

2. Terminal 2: Start Whisper GUI
   $ cd ~/Documents/whisper && python whisper_gui.py

3. GUI detects process:
   - Scans processes, finds "claude (PID: 9012)"
   - Displays in dropdown: "None" | "claude (PID: 9012)"

4. User selects "claude (PID: 9012)" from dropdown

5. User clicks "Start Recording"

6. Whisper records audio

7. User clicks "Stop Recording"

8. Whisper transcribes using OpenAI model

9. Whisper sends transcription:
   a. Writes to /proc/9012/fd/0 (Claude's stdin)
   b. Copies to clipboard (Ctrl+V fallback)
   c. Adds to history in GUI

10. Claude receives text in its input and processes it

================================================================================
COMMON ISSUES & SOLUTIONS
================================================================================

Issue: GUI doesn't detect processes
  Fix: Check process is running with 'claude' or 'codex' in its command line
       Try: grep -l 'claude\|codex' /proc/*/cmdline

Issue: stdin injection fails
  Fix: Ensure target process has accessible stdin: ls -la /proc/[PID]/fd/0
       Some TTY configurations may prevent writing to /proc/[PID]/fd/0
       Fallback: Use clipboard instead

Issue: ALSA/JACK warnings
  Fix: Use wrapper script: ./whisper
       Or suppress: python whisper_cli.py 2>/dev/null

Issue: Hotkey daemon doesn't work
  Fix: Ensure running from real desktop terminal (not SSH/IDE)
       Check: DISPLAY or WAYLAND_DISPLAY environment variable
       Run: check_real_session.py to diagnose

================================================================================
PROCESS STATE MACHINE
================================================================================

INIT
 ├─ Load Whisper model (GPU intensive, one-time)
 ├─ Initialize PyAudio
 └─ Load/configure microphone

READY (can start recording)

RECORDING (audio frames being collected)

TRANSCRIBING (Whisper AI processing)

OUTPUT (writing to stdin/FIFO/clipboard)

└─ Back to READY (loop) OR CLEANUP (exit)
